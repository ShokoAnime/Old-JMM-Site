Installing and Setting Up SQL Server

JMM supports Microsoft SQL Server 2008, and we recommend it (or MySQL) for users who have large collections or just want the fastest possible performance. SQL Server should only be used by advanced users, we recommend MySQL for users looks for a performance boost without a complicated setup.

Why SQL Server Over SQLite

If your collection contains more than 200 series, performance wise your better off using SQL Server. While SQLite is a one click setup single file, it's not designed to handle a database that size and it will show it's limitations as your collection continues to grow by causing slowdowns when accessed by JMM.

Some of the differences between SQL Server and SQLite

  • SQL Server is more powerful but will use more memory than SQLite.
  • SQLite does not handle multiple operations as well as SQL Server.
  • Using SQLite and running any kind of tasks in JMM Server will cause considerable and noticeable slowdowns when running multiple JMM products.

Download

SQL Server is available in many different versions, for use with JMM Server you'll want to download SQL Server Express 2008 which is free to download.

Installing SQL Server

Start the Installer and then go to the Installation section. From there click on New SQL server standalone installation.

The installer will check for setup dependencies. If you receive no errors click OK to continue, if you do receive any errors they need to be fixed before you can proceed further with the install.

Follow the instructions and proceed through the product key and license page. Click the Install button on the Setup Support Files section.

Configuring SQL Server

Once complete, select the following options.

In the Database Engine Configuration you'll have the option of setting the Administrator password. If you already know how to set up users in SQL Server you can skip this step, otherwise select Mixed Mode and enter a password. The username for this password is sa as this is the Username and Password for JMM Server setup. Make sure you press the Add Current User button.

From here you can click through on the Next button to finish the installation.

Configuring JMM Server

If you're running JMM Server for the first time you'll be prompted to select a database type, select SQL Server.

If you're changing the database type to SQL Server click the Setup tab and select SQL Server.

You'll be asked to input the following settings.
Server Address - Only change if connection to a database on a different computer. Default: Localhost
Database Name - Only change if created a custom database for JMM. Default: JMMServer
Username - If you created an account for JMM, use that username. Default: root
Password - If you created an account for JMM, use that username, otherwise it's the root password you inputted.

Pressing the Refresh button will get a list of computers running SQL Server on your local network.

If successful JMM Server will connect, create the database (initial connect only) and populate it.

You've successfully configured JMM Server with SQL Server!
