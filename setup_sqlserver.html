SQL Server Setup

JMM supports Microsoft SQL Server 2008, and we recommend it (or MySQL) for users who have large collections or just want the fastest possible performance.

Why SQL Server Over MySQL

If your collection contains more than 200 series, performance wise your better off using either SQL Server or MySQL. While SQLite is a one click setup single file, it's not designed to handle a database that size.

Some of the differences between SQL Server and SQLite

  • SQL Server is more powerful but will use more memory than SQLite.
  • SQLite does not handle multiple operations as well as SQL Server.
  • Using SQLite and running any kind of tasks in JMM Server will cause considerable and noticeable slowdowns in JMM Desktop, My Anime 3, or Plex Agent when running them at the same time.

SQL Server should only be used by advanced users. We recommend MySQL for users looks for a performance boost without a complicated setup.

Install

Start the Installer and then go to the Installation section.

The installer will check for setup dependencies. If you receive no errors click OK to continue, if you do receive any errors you must fix them first before proceeding on.

Follow the instructions and proceed through the product key and license page. Click the Install button on the Setup Support Files section.

Once complete, select the following options.

In the Database Engine Configuration you will have the option of setting the Administrator password. If you already know how to set up users in SQL Server you can miss this step. Otherwise select Mixed Mode and enter a password. The username for this password is sa as this is the Username and Password for JMM Server setup.

Make sure you press the Add Current User button.

From here you can click through on the Next button to finish the installation.

Run JMM Server Setup

If you are running JMM Server for the first time it will prompt you to select a database type. Otherwise you can go to the setup tab and enter your details there. Enter all your details from above and press the Save and Start Server button.

Th server name will be localhost if you are running JMM Server on the same machine as SQL Server. Alternatively you can enter the name of the server yourself, or press the Refresh button to get a list of machines running SQL Server on your local network.

Once this is successful, you can go to the Import Folders tab and start adding your collection
