Import Folders Tab

The Import Folders tab is where you'll manage all your import locations for your files. At least one folder is required before you can start using any JMM program or plugin.

Click New Folder to add a import folder for your collection.

A new window will open allowing you to input the location of your files, this can either be the physical location or a networked location. We recommend clicking the folder icon to select the location to avoid any mistakes from manually typing the location in, especially if you're mapping a networked location. Additionally, if anything changes on your network, you'll need to update the location of any import folders that use a networked location or you'll be presented with errors.

You'll also be presented with three different options that will tell JMM Server what to do with this import folder.

  • Drop Source - If selected, JMM Server will monitor this folder and move any files it detects to the folder selected as a Drop Destination. Additionally JMM Server will automatically group files that belong to the same series and create the series folder in the Drop Destination if it doesn't already exist. You can have as many Drop Sources as you want, however you must setup a Drop Destination to use Drop Source.
  • Drop Destination - Used in conjunction with Drop Source, JMM Server will move all files from your Drop Source/Drop Sources into the selected Drop Destination. You can only have one folder listed as a Drop Destination.
  • Watch For New Files - If you don't want JMM Server moving your files and instead just want it to watch a specified folder then select Watch For New Files. JMM Server will monitor the selected folder and add any files it detects to your collection.

How Long Will Importing Take

Every file you add needs to be hashed before it can be added to your collection and while the time it takes to do this will vary depending on the computer setup, mainly the CPU and HDD read/write speed. The average computer should take 1 hour to process 80GB worth of files.

Hashing benchmarks on External USB 2.0 Drive
File Size: 520MB

Using Hasher.dll
ED2k Hash only = 18 seconds
ED2k+CRC32+MD5+SH1 = 24 seconds

NOT Using Hasher.dll
ED2k Hash only = 20 seconds
ED2k+CRC32+MD5+SH1 = 28 seconds

Hashing benchmarks on Internal Drive
