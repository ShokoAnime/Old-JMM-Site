File Playback

With Anime Buddy, you can connect JMM Server allowing to locally and remotely playback the files in your collection. Additionally with Anime Buddy, your watched state data will be automatically tracked and updated in your collection. You can also manually set the watched state for files you've watched outside of Anime Buddy.

After installing Anime Buddy, click Login to JMM Server in the upper-right hand corner. You'll be taken to the following menu.

You'll be asked to input some information Anime Buddy needs to connect to JMM Server.

If JMM Server and Anime Buddy are on the same computer, use the following information

JMM Server Address: Use either 127.0.0.1 or localhost
JMM Server Port: By default it's 8111
Username: Input your JMM Desktop Username
Password: Input your JMM Desktop Password, if blank than leave blank

If JMM Server and Anime Buddy are on different computers, use the following information.

JMM Server Address: Input the IP address of the computer JMM Server is on
JMM Server Port: By default it's 8111
Username: Input your JMM Desktop Username
Password: Input your JMM Desktop Password, if blank than leave blank

If everything works by default then you're all set! However for most people some additional steps will need to be taken to get Anime Buddy to connect to JMM Server.

JMM Server and Anime Buddy on the same computer

By design, Windows prevents Windows Store Apps from sending network traffic to a local machine. You can override this and get a more detail explanation from the following link.

JMM Server and Anime Buddy on different computers (Same Network)

If you don't know how to get the IP Address of the computer JMM Server is on visit the following site on the computer JMM Server is on and write down the IP it shows you.

  • Adding JMM Server and Anime Buddy to Windows Firewall Exception List
  • Creating An Inbound Rule for JMM Server

You'll need to add an exception for both JMM Server and Anime Buddy on the computer where JMM Server is installed. If you don't have Anime Buddy installed than install it before proceeding.

These instructions might vary between different versions of Windows.

Open up the Control Panel and select Network and Internet, then select Network and Sharing Center. In the bottom left-hand corner select Windows Firewall and then from the menu on the right select Allow an app or feature through Window Firewall, a list of programs will open.

Anime Buddy will be there by default, enable the Private and Public exception for each entry if there are multiple entries. If JMM Server is not there you'll need to add it, click the allow another app button and navigate to where you've installed JMM Server. After adding JMM Server enable the Private and Public exception for JMM Server.

Open Anime Buddy and try to connect to JMM Server again, if it works then you're set. If not then you'll proceed to the next tab.

Because we're going to be adding an Inbound Rule to Windows Firewall which for some might be a little too advanced, we'll be doing this step-by-step with images showing every part.

In the Windows Firewall settings, click Advanced Settings.

Windows will open up a new window with a lot of information and options, for our purposes we need to add a new inbound rule. First make sure Inbound Rules is selected. On the right side is the Actions menu, from there select New Rule... to create a new inbound rule.

Another window will open allowing us configure or new inbound rule. Since were adding a port, select Port for the Rule Type and then click Next.

Protocol and Ports allows us to select the port type and the port itself, select TCP for the first option and select Specific local ports for the second option. In the box you'll need to input the port JMM Server uses, by default it's 8111.Once done click Next.

Now we need to set the Action for our rule, since we want to be able to connect select Allow the connection and click Next.

The Profile tells Windows how to apply our inbound rule, select Domain and Private but not Public and then click Next.

Almost done! Now we just need to name our inbound Rule, you can put whatever you want as it doesn't matter. For this example I put JMM Server - Anime Buddy, add a description of you'd like and once done click Finish to have your inbound rule added.

You should now be able to connect to JMM Server with no problem.

JMM Server and Anime Buddy on different computers (Different Network)

First you'll need to open the port JMM Server uses. Use PortForward to get specific instructions on how to enable port forwarding on your router. Once down check and make sure the port is actually opened and accessible over the internet. If you get a success message then you should be able to connect, if you get a failure message than continue reading.

The most common reason you get a failure message even after opening the port is due to a security feature most routers have built in called Port Scan and DoS Protection, how to access this setting varies between routers. If you can add an exception to it, add the port JMM Server uses, if not then you'll need to disable it. We advise you do your own research before disabling it so you know what it does and if you need it, however it has been known to cause issues for just about every device that connects over the interent.

If you've inputted the correct information, Anime Buddy will refresh and your collection will be shown.
