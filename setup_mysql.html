MySQL Setup

JMM supports MySQL, and we recommend it (or SQL Server 2012) for users who have large collections or just want the fastest possible performance.

Database Recommendations

If you have more than 200 series in your collection you will be much better off using SQL Server or MySQL (compared to sqlite). However sqlite is just a single file, which means you can move it around much more easily.

The installation process for MySQL is much simpler compared to SQL Server. SQL Server will use a lot more memory than sqlite.

Sqlite does not handle multiple operations as well as SQL Server or MySQL. So if you have more than one user connecting to your JMM Server, sqlite will be much slower. Sqlite will also perform much slower if you are trying to use JMM Desktop or MyAnime3 when JMM server is importing files or running other processeses.

Install

The following instructions refer to version 5.6.20.0. However, any newer versions should be similar

Start the installer, and download any updates

Select the 'Custom' install as below

Select the options as shown

Check the pre-requisites and then click the 'Next' button

Follow through with the installation, until you get to the configuration screen

Choose the configuration type that best suits your needs

Set the root (Administrator) login details

root is default username in JMM Server setup and Password will also be needed there as well.

Alternatively you can create a new MySQL user specifically for use by JMM Server

Optional – Create a database

JMM Server will automatically create a database for you, but if you prefer to create it yourself, you should do it at this stage.

Run JMM Server Setup

If you are running JMM Server for the first time it will prompt you to select a database type. Otherwise you can go to the setup tab and enter your details there.

Enter all your details from above and press the ‘Save and Start Server’ button.

Once this is successful, you can go to the ‘Import Folders’ tab and start adding your collection
