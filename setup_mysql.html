MySQL Setup

MySQL is the reconmmend database for use with JMM Server for users with a large collection or who intend to build a large collection or simply for any user that wants better and faster performace with JMM.

Why MySQL over SQLite

If your collection contains more than 200 series, performance wise your better off using either SQL Server or MySQL. While SQLite is a one click setup single file, it's not designed to handle a database that size.

Some of the differences between MySQL and SQLite

  • MySQL is more powerful but will use more memory than SQLite.
  • SQLite does not handle multiple operations as well as MySQL.
  • Using SQLite and running any kind of tasks in JMM Server will cause considerable and noticeable slowdowns in JMM Desktop, My Anime 3, or Plex Agent when running them at the same time.

The installation process for MySQL is much simpler compared to SQL Server.

Install

The following instructions refer to version 5.6.20.0. While newer versions should not be the different, pay attention to the items being install as that's whats important.

Start the installer, and download any updates. Select the Custom install option.

Select the options as shown in the following images. In some versions of the installer you're presented with both a 32bit and 64bit option, install the one releated to your setup, if unsure you can install both.

Check the pre-requisites and then click the Next button.

The installer will automatically download and install the selected products, just wait until it's done. If anything fails click the retry button until it succeeds.

Choose the configuration type that best suits your needs

Set the Administrator login details. JMM Server uses root as the default username, a password is required for JMM Server. Alternatively you can create a new MySQL user specifically for use by JMM Server.
